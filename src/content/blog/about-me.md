สวัสดีครับ ผมเบิร์ด

ผมเป็น software engineer คนหนึ่งที่ค้นพบว่าตัวเองมีความสุขกับการ "ทำของ"
มากกว่าการพูดถึงมัน ไอเดียที่ค้างอยู่ในหัวมันไม่เคยมีน้ำหนักเท่าของที่ทำเสร็จ
แล้วมีคนได้ใช้จริง — เพราะงั้นผมเลยชอบเขียนให้เสร็จ ปล่อยออกไป เห็นผล
แล้วค่อยกลับมาปรับ มากกว่าจะนั่งขัดเกลามันอยู่ในที่มืดเป็นเดือน

## กลางวัน

กลางวันผมทำงานฝั่ง enterprise — ระบบหลังบ้านที่คนเยอะๆ พึ่งพา
เป็นงานที่ได้เล่นกับสถาปัตยกรรมจริงจัง อย่าง clean และ hexagonal architecture,
การค่อยๆ ย้ายระบบเก่าจาก Java ไป Go ทีละชิ้นโดยไม่ทำให้ของเดิมพัง,
และการมองทั้ง stack ให้ออกว่าเวลามีอะไรช้าหรือพัง มันเกิดตรงไหน

งานแบบนี้สอนผมเรื่องหนึ่งที่ติดตัวมาตลอด คือ **อย่าเพิ่งเชื่อจนกว่าจะพิสูจน์**
ไม่เดาว่าบั๊กอยู่ตรงไหนจากชื่อไฟล์ ไม่บอกว่า "เสร็จแล้ว" จนกว่าจะได้รันดูจริงๆ
การเปิดดูของจริงหนึ่งครั้ง ถูกกว่าการเดาผิดแล้วต้องถอยกลับมาทำใหม่เสมอ

## กลางคืนกับวันหยุด

พอหมดเวลางาน ผมก็ยังสนุกกับการสร้างของอยู่ดี แต่เป็นของของผมเอง:

- **ระบบลงทะเบียนค่ายเด็กให้มูลนิธิ** — ให้พ่อแม่ล็อกอินด้วย LINE,
  จัดการข้อมูลเด็กกับผู้ปกครอง, ออกรายงานการเงินให้ทีมงาน
  สิ่งที่ผมแคร์ที่สุดในโปรเจกต์นี้คือ พ่อแม่ที่ไม่ได้สาย tech ต้องกดใช้บนมือถือ
  ได้แบบไม่ต้องคิด
- **AI memory framework ของตัวเอง** — เครื่องมือที่ทำให้ AI ที่ผมทำงานด้วย
  จำสิ่งที่เคยทำข้าม session ได้ ไม่ต้องเริ่มจากศูนย์ทุกครั้ง
- **รัน LLM บนเครื่องตัวเอง** — เพราะผมอยากเข้าใจมันถึงราก
  ไม่ใช่แค่เรียก API แล้วจบ

## สิ่งที่ผมยึดไว้

ผ่านงานมาหลายแบบ สุดท้ายมันเหลือหลักไม่กี่ข้อที่ผมเชื่อจริงๆ:

- **ของต้องง่าย** — ไม่ over-engineer ไม่สร้าง abstraction ก่อนถึงเวลาที่ต้องใช้มันจริง
- **UX มาก่อนเสมอ** — โค้ดจะสวยแค่ไหน ถ้าคนใช้แล้วงง มันก็เปล่าประโยชน์
- **Verify ก่อนพูด** — พิสูจน์ก่อน แล้วค่อยสรุป
- **ไปให้ยาว ไม่ใช่ไปให้เร็ว** — งานที่ดีคืองานที่ทำต่อไปได้เรื่อยๆ
  ครอบครัวกับการพักผ่อนมาก่อน ไม่เผาตัวเองเพื่อ sprint เดียว

บล็อกนี้ผมตั้งใจเขียนไว้บันทึกสิ่งที่เรียนรู้ระหว่างทาง
ส่วนใหญ่เป็นเรื่องที่ผมเพิ่งพลาดมาเอง แล้วไม่อยากพลาดซ้ำ —
เผื่อมันจะเป็นประโยชน์กับคนที่เดินมาทางเดียวกันบ้าง

ขอบคุณที่แวะมาอ่านครับ 🙂
