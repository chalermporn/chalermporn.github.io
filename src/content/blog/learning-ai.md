ช่วงหลังผมทำงานกับ AI แทบทุกวัน ทั้งใช้มันช่วยเขียนโค้ด
ทั้งสร้างเครื่องมือของตัวเองรอบๆ มัน นี่คือบทเรียนไม่กี่ข้อที่ตกผลึกแล้วจริงๆ

## 1. โมเดลเล็กชอบ "เออออ" ตามที่เราถาม

เวลาให้ AI ตัวเล็กไปสรุปหรือยืนยันอะไร มันมักจะตอบในทางที่เราอยากได้ยิน
ถ้าถามว่า "อันนี้ใช่ไหม" มันก็มีแนวโน้มจะบอกว่า "ใช่"
ทั้งที่ความจริงอาจไม่ใช่

ผมเลยตั้งกฎกับตัวเองว่า **คำตอบจากโมเดลไม่นับเป็นหลักฐาน**
ถ้าจะยืนยันว่า API ตัวนี้มีจริงไหม เวอร์ชันนี้ออกหรือยัง
ต้องไปยิง raw API ดู status code จริงๆ ไม่ใช่ถามโมเดลแล้วเชื่อ

## 2. ความจำของเราเองก็ล้าได้

พอโลกของ AI ขยับเร็วมาก ความรู้ที่เคยจำมาก็มีวันหมดอายุ
หลายครั้งที่ผมเกือบจะ "แก้" ชื่อเวอร์ชันหรือชื่อโมเดลที่ผู้ใช้พิมพ์มา
เพราะคิดว่ามันพิมพ์ผิด — ทั้งที่จริงๆ มันคือของใหม่ที่ออกมาหลังจากที่ผมรู้

บทเรียนคือ เวลาเจอชื่อที่ "รู้สึกว่าไม่มี" ให้ **probe ดูจริงก่อน**
อย่าเพิ่งด่วนสรุปว่าอีกฝ่ายพิมพ์ผิด สมมติว่าเราเองนั่นแหละที่ตกข่าว

## 3. ให้ AI จำงานข้าม session ได้ เปลี่ยนทุกอย่าง

ปัญหาที่น่าหงุดหงิดที่สุดของการทำงานกับ AI คือมันลืมทุกอย่างเมื่อปิดหน้าต่าง
ผมเลยลองสร้างระบบความจำให้มัน — เก็บสิ่งที่เคยทำ เคยตัดสินใจ เคยพลาด
ไว้เป็นไฟล์ แล้วให้มันค้นเจอด้วย vector search ตอนเริ่มงานใหม่

ผลคือมันไม่ต้องเริ่มจากศูนย์ทุกครั้ง และที่สำคัญกว่าคือ
มัน "จำ" บทเรียนที่เคยสอนไปแล้วได้ ไม่ต้องสอนซ้ำเรื่องเดิมทุกวัน

## สรุป

ทั้งสามข้อมันมารวมที่หลักเดียวกัน — **อย่าเชื่อ จนกว่าจะพิสูจน์**
ไม่ว่าจะเป็นคำตอบของโมเดล หรือความมั่นใจของตัวเราเอง
การเปิดดูของจริงหนึ่งครั้ง ถูกกว่าการเดาผิดแล้วเดินต่อเสมอ
